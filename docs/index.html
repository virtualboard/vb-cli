Why VirtualBoard CLI?

Workspace Bootstrap

Initialise `.virtualboard/` with curated templates and metadata using vb init, no git dependencies required.

Feature Lifecycle

Create, update, move, delete, and lock feature specs with purpose-built commands that respect workflow guardrails.

Validation + Indexing

Enforce schema, workflow, and dependency rules with vb validate. Regenerate Markdown, JSON, and HTML indexes via vb index.

Automation Friendly

JSON output, dry-run support, and log streaming make it easy to plug VirtualBoard into CI/CD pipelines and bots.

Quick Start

Install the CLI and explore the workspace helpers in minutes.

go install github.com/virtualboard/vb-cli@latest
vb --help

# From your repository root
vb init
vb new "Awesome Feature" label1 label2
vb validate all
vb index

All commands accept --json, --dry-run, --verbose, and --root flags to adapt to your tooling.

Command Highlights

Initialize

vb init downloads and expands the VirtualBoard template archive. Use --force to rebuild an existing workspace.

New Feature

vb new scaffolds a spec with labels, front-matter, and canonical sections ready for backlog grooming.

Move

vb move enforces workflow transitions and can assign owners in one command.

Update

vb update patches fields or body sections to keep specs synchronised with product decisions.

Validate

vb validate surfaces schema, dependency, and workflow issues early. Add --fix to reapply templates.

Index & Lock

vb index rebuilds stakeholder-friendly views, while vb lock coordinates collaborative edits.

ASCII Art

vb art renders the VirtualBoard logo as beautiful colored ASCII art in your terminal.
